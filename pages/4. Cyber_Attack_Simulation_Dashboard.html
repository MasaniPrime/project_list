Cyber Attack Simulation Dashboard

Interactive visualization of common cyber threats and defense strategies

Cybersecurity Data Visualization Simulation Python Flask SQLite JavaScript

Project Overview

Cyber Attack Simulation Dashboard — an interactive tool that simulates common cyber threats (brute-force attacks, phishing attempts, ransomware spread) and visualizes them in real time. The dashboard helps demonstrate how attacks unfold, their impact, and potential defenses. This project is highly engaging for networking because it combines cybersecurity awareness, data visualization, and simulation modeling — all achievable from a laptop.

Key Considerations and Goals

  • Primary goal: Create a dashboard that simulates different attack scenarios and shows their progression visually.
  • Secondary goals: Provide educational context (attack type, detection methods, mitigation strategies), and allow users to toggle between scenarios.
  • Constraints: Laptop-only build; use simulated data (no real attacks); focus on visualization and awareness.
  • Success criteria: Smooth interactive dashboard, clear attack progression visuals, and educational explanations.

Tech Stack and Tools

  • Python (simulation logic + backend)
  • Flask / FastAPI (API backend)
  • JavaScript + D3.js / Plotly (visualizations)
  • SQLite (store simulated logs)
  • NetworkX (simulate network graphs for ransomware spread)
  • Matplotlib / Seaborn (basic charts)

Step‑by‑Step Build Guide

Phase 0: Planning (1–2 days)

Define attack scenarios:

  1. Brute-force login attempts (failed vs successful).
  2. Phishing campaign (emails sent, clicked, compromised).
  3. Ransomware spread (infected nodes in a network graph).

Decide on visualization style (line charts, counters, animated graphs).

Phase 1: Simulation Logic (2–3 days)

  • Write Python scripts to generate synthetic logs for each attack type.
  • Example: Brute-force → generate 100 login attempts with random success/failure.
  • Store logs in SQLite for easy retrieval.

Phase 2: Visualization (3–4 days)

Build interactive charts:

  • Line chart showing brute-force attempts over time.
  • Funnel chart for phishing (emails sent → clicked → compromised).
  • Network graph showing ransomware spread across nodes.

Use D3.js or Plotly for dynamic visuals.

Phase 3: Dashboard Integration (2–3 days)

  • Create a Flask/FastAPI backend to serve simulated data.
  • Build a simple frontend with buttons to toggle scenarios.
  • Display attack type, progression, and mitigation tips alongside visuals.

Phase 4: Educational Layer (2–3 days)

Add text explanations for each scenario:

  • What the attack is.
  • How it's detected.
  • Common defenses.

Provide "Mitigation Tips" panel (e.g., strong passwords, MFA, employee training).

Phase 5: Demo and Deployment (2–4 days)

  • Host locally or on Heroku/Streamlit for demo.
  • Prepare 3 demo runs (brute-force, phishing, ransomware).
  • Record a short demo video showing attack progression.

Data Sources and Evaluation

  • Synthetic data: Generate logs programmatically (no real attacks).
  • Evaluation:
    • Test dashboard responsiveness with 1,000+ simulated events.
    • Ensure visuals update smoothly.
    • Validate educational content with cybersecurity best practices.

Demo and Networking Tips

  • Pitch line: "I built a dashboard that simulates cyber attacks in real time — you can see how brute-force, phishing, and ransomware unfold and how to defend against them."
  • Demo: Toggle between scenarios, show attack progression, highlight defenses.
  • Networking impact: Visually striking, educational, and demonstrates cybersecurity + visualization skills.
