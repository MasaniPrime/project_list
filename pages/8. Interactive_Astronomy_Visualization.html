Interactive Astronomy Visualization

A web-based tool that brings the cosmos to life by visualizing planetary systems, star maps, and galaxies in interactive 3D. Users can explore the universe, zoom into planetary orbits, filter by properties like habitability, and discover fun astronomical facts in real time.

Data Visualization 3D Graphics Interactive Design Geospatial Data Three.js D3.js JavaScript Python APIs GitHub Pages

Overview

This project transforms astronomy into an interactive experience. Instead of static diagrams, users explore real planetary systems, zoom into star maps, and toggle overlays showing temperature, brightness, and habitability. The visualization combines NASA/ESA data with Three.js for 3D rendering and D3.js for star maps, creating a stunning, educational interface.

Goals

  • Goal 1: Visualize planetary systems and star maps using 3D graphics and interactive controls.
  • Goal 2: Integrate NASA Exoplanet Archive and ESA Gaia DR3 data into the visualization.
  • Goal 3: Implement filters and overlays for exploring exoplanets by properties (Earth-like, habitable, etc.).
  • Goal 4: Create an educational layer with scientific details and fun facts.
  • Goal 5: Deploy an accessible, performant visualization that handles 1,000+ stars smoothly.

Tech Stack

  • Frontend: HTML5, CSS3, JavaScript
  • 3D Visualization: Three.js (for orbital mechanics and planetary systems)
  • 2D Visualization: D3.js (for star maps and constellation overlays)
  • Backend (optional): Python Flask for data preprocessing and serving APIs
  • Data Sources: NASA Exoplanet Archive, ESA Gaia DR3, Open Astronomy APIs
  • Deployment: GitHub Pages, Heroku, or static hosting

Step-by-Step Build Plan

Phase 0: Planning (1–2 days)

Define scope and core features:

  • Planetary system visualization with zoomable orbits
  • Star map with constellation overlays
  • Galaxy overview or Milky Way visualization

Example features to implement:

  • Zoomable star map with constellations
  • Planetary system view with orbits, distances, and relative sizes
  • Toggle overlays (temperature, brightness, habitability index)

Phase 1: Data Collection (2–3 days)

  • Download datasets from NASA Exoplanet Archive
  • Fetch star data from ESA Gaia DR3 (positions, brightness, distances)
  • Normalize all data into standardized JSON format
  • Validate data accuracy and coverage

Phase 2: Visualization Setup (3–4 days)

  • Set up Three.js scene with camera, lighting, and renderers
  • Implement planetary orbit mechanics (calculate positions over time)
  • Use D3.js for 2D star map and constellation overlays
  • Implement zoom, pan, and rotate controls
  • Add hover tooltips for stars and planets

Phase 3: Interactive Features (2–3 days)

  • Add filter controls (show only Earth-like planets, habitable zone, etc.)
  • Implement overlay toggles (temperature heatmap, brightness levels, habitability index)
  • Build search bar for finding specific stars or exoplanets
  • Add preset views (Solar System, Alpha Centauri, nearest exoplanets)

Phase 4: Educational Layer (2–3 days)

  • Create info panels displaying planet/star details (mass, radius, distance, age)
  • Add fun facts (e.g., "This star is 10x brighter than the Sun")
  • Implement toggle for "scientific mode" vs "casual mode"
  • Add comparison tools (e.g., "Compare this planet to Earth")

Phase 5: Deployment and Demo (2–4 days)

  • Host on GitHub Pages or Heroku
  • Prepare demo datasets (Milky Way star cluster, Proxima Centauri system, habitable exoplanets)
  • Record demo video showing zooming into a planetary system and toggling overlays
  • Optimize performance for web browsers

Data Sources and Evaluation

Data Sources:

  • ESA Gaia DR3 - Star positions, brightness, distances
